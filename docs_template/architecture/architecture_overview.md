# Architecture Overview

## Runtime Architecture

This section describes the runtime components of the system, how they are deployed on hardware nodes, and how these components connect with each other. If possible, we also describe the data flows and control flows between components, corresponding to some important interaction with the system. 

## Module Architecture

This section describes the code modules making up the runtime components of the system. It provides a map for developer to understand the structure of the codebase and know where a certain feature of the codebase should be added or modified.
